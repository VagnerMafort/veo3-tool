2. Sistema de Usuários

2.1 Autenticação

Login por email/senha com Flask-Login. Senhas hasheadas com Werkzeug (PBKDF2). Rate limit: 5 tentativas/min no login, 3/min no cadastro, 2/5min no esqueci senha.

2.2 Níveis de Acesso

Admin Master{{ '{{ ADMIN_MASTER_EMAIL }}' }} — pode conceder/remover admin
AdminAcesso ao painel admin, créditos ilimitados
Usuário com planoAcesso às funcionalidades conforme plano
Usuário sem planoBloqueado — sem acesso a APIs

2.3 Isolamento de Dados

Todos os arquivos são nomeados com user_id como prefixo. Todas as queries filtram por user_id. Rotas de arquivo verificam propriedade. Path traversal bloqueado com normpath().

3.3 Lógica de Cobrança

Renderização e legenda são grátis se o vídeo já gastou créditos com outros serviços (imagens, narração, animação). Plano API Própria e Admin não gastam créditos. Créditos de animação são cobrados só pelas cenas que deram certo.

3.4 Stripe

4. Módulo de Vídeo

4.1 Fluxo de Criação

1. Usuário escreve roteiro → 2. IA divide em cenas (GPT-4o-mini) → 3. Gera imagens por cena (gpt-image-1) → 4. Usuário revisa storyboard → 5. Configura narração/animação/legendas → 6. Gera vídeo final (FFmpeg)

4.2 Narração

MiniMax Speech-02-HD. Texto completo narrado de uma vez. Whisper transcreve pra sincronizar timestamps com cenas. 16 vozes padrão + clonagem de voz.

4.3 Animação

MiniMax Hailuo 2.3-Fast. Image-to-video, 6s por clipe, 768P. Processamento em lotes de 3. Cenas do banco são reutilizadas sem gerar. Detecção de saldo insuficiente vs rate limit.

4.4 Montagem Final

FFmpeg concatena clipes + áudio. Legendas SRT palavra por palavra via Whisper. Música de fundo mixada em volume baixo. Zoompan em imagens estáticas. Suporte a mix de imagens e vídeos do banco.

5. Thumbnail Engine

5.1 Geração do Zero

IA analisa roteiro → identifica cena mais clicável → gera 4 variações com composições diferentes (close-up, medium, low angle, wide). 10 estilos: Cinematic, Hyper Realistic, Mystery, Documentary, Cartoon, Kids, Dark Drama, Sci-Fi, Bright Viral, Minimal. Pós-processamento: sharpen + contraste + brilho.

5.2 Edição por IA

Upload de thumbnail + instrução textual. Suporte a troca de rosto/personagem. Usa endpoint /v1/images/edits do gpt-image-1. Preserva composição original.

5.3 Viral Tuning

Melhoria automática: mais contraste, nitidez, cores fortes, fundo limpo, destaque no sujeito. Comparação antes/depois.

5.4 Ranking

GPT-4o-mini analisa thumbnails com visão. Score 0-100 baseado em: contraste, clareza, foco, tamanho do rosto, emoção, legibilidade, separação fundo/sujeito.

5.5 Editor Visual

Fabric.js. Texto editável (fonte, cor, contorno, sombra, espaçamento). Seta, círculo, retângulo, linha. Upload de imagens extras. Filtros (brilho, contraste, blur). Camadas com ordenação. Undo/redo (30 estados). Export 1280×720 e 2048×1152. Salvar projeto em JSON. Integração com IA (editar/viral tuning direto do editor). Salvar na biblioteca.

6. Segurança

6.1 Autenticação e Autorização

Flask-Login com @login_required em todas as rotas protegidas. Admin verificado com is_admin. Admin Master verificado por email. Senhas PBKDF2.

6.2 Rate Limiting

Login5 tentativas/min por IP
Cadastro3 tentativas/min por IP
Esqueci senha2 tentativas/5min por IP
Gerar storyboard3/min por usuário
Regerar cena5/min por usuário
Gerar thumbnails3/min por usuário

6.3 Proteções

Path traversal: normpath() + verificação de prefixo em todas as rotas de arquivo. Upload: limite de tamanho (20MB imagens, 30MB áudio, 50MB música, 5MB projetos JSON). Webhook Stripe: signature verification obrigatória. API keys: só acessíveis com plano ativo.

8. Manutenção

8.1 Deploy

cd /var/www/veo3-tool && git pull origin main && sudo systemctl restart veo3

8.2 Logs

sudo journalctl -u veo3 -f

8.3 Cleanup Automático

Arquivos temporários em outputs/ e storyboards/ são removidos automaticamente após 7 dias na inicialização do servidor.

8.4 Backup do Banco

cp /var/www/veo3-tool/instance/veo3.db /backup/veo3_$(date +%Y%m%d).db

8.5 Nginx

Config em /opt/mordomo/nginx.conf (dentro do Docker mordomo-nginx). SSL via Certbot no container mordomo-certbot. Reload: docker exec mordomo-nginx nginx -s reload

8.6 Serviço Systemd

sudo systemctl status veo3
sudo systemctl restart veo3
sudo systemctl stop veo3
